Every extraction response returns structured JSON matching your schema, plus citations showing where data came from.

## Response structure

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "extracted_data": {...},
    "citations": [...]
  }
}
```

### Top-level fields

| Field | Type | Description |
|-------|------|-------------|
| `code` | int | HTTP status code (200 = success) |
| `message` | string | Status message |
| `data` | object | Extraction results |

---

## Extracted data

JSON matching your schema structure:

```json
{
  "extracted_data": {
    "invoice_number": "INV-2024-001",
    "invoice_date": "2024-01-15",
    "vendor_name": "Acme Corp",
    "total_amount": 1250.00,
    "line_items": [
      {
        "description": "Product A",
        "quantity": 5,
        "price": 250.00
      },
      {
        "description": "Product B",
        "quantity": 2,
        "price": 500.00
      }
    ]
  }
}
```

**Field behavior:**
- Missing or unextractable fields return `null`
- Arrays return empty `[]` if no matches found
- Numbers are parsed as `float` or `int` based on schema type
- Enums return the exact value from your predefined list

---

## Citations

Source locations for each extracted field. Citations are returned as an **object** keyed by field name:

```json
{
  "citations": {
    "invoice_number": {
      "value": "INV-2024-001",
      "bounding_regions": [
        {
          "page_number": 1,
          "position": [137, 599, 1129, 599, 1129, 625, 182, 625],
          "text": "INV-2024-001"
        }
      ]
    },
    "total_amount": {
      "value": "1250.00",
      "bounding_regions": [
        {
          "page_number": 2,
          "position": [812, 802, 878, 802, 878, 843, 812, 843],
          "text": "1250.00"
        }
      ]
    }
  }
}
```

### Citation structure

Each field in `citations` contains:

| Field | Type | Description |
|-------|------|-------------|
| `value` | string | Extracted value for this field |
| `bounding_regions` | array | Array of regions where this field was found |

### Bounding region fields

| Field | Type | Description |
|-------|------|-------------|
| `page_number` | int | Page number (starts at 1) |
| `text` | string | Original text from document |
| `position` | array | Four-point bounding box in **pixel coordinates** `[x1, y1, x2, y2, x3, y3, x4, y4]` |

**Use for:**
- Verifying extraction accuracy
- Highlighting fields in UI
- Building audit trails
- Visual grounding

---

## Coordinates

Bounding boxes use four-point representation with eight values in **pixel coordinates**:

```
position: [x1, y1, x2, y2, x3, y3, x4, y4]
```

The four points follow this order:

```
(x1,y1) -------- (x2,y2)
   |                |
   |  Field Area    |
   |                |
(x4,y4) -------- (x3,y3)
```

**Coordinate format:**
- Eight integer values representing pixel positions
- Points ordered: top-left, top-right, bottom-right, bottom-left
- Example: `[137, 599, 1129, 599, 1129, 625, 182, 625]`

### Use coordinates

```python
# Extract position from citation
citation = result.citations["invoice_number"]
position = citation["bounding_regions"][0]["position"]

# Get four corner points (already in pixels)
x1, y1 = position[0], position[1]  # Top-left
x2, y2 = position[2], position[3]  # Top-right
x3, y3 = position[4], position[5]  # Bottom-right
x4, y4 = position[6], position[7]  # Bottom-left

# Calculate bounding box dimensions
width = x2 - x1
height = y3 - y1
```

## Error handling

When extraction fails:

```json
{
  "code": 40004,
  "message": "Parameter error: Invalid schema structure"
}
```

### Common errors

| Code | Description | Solution |
|------|-------------|----------|
| 40004 | Invalid schema | Check schema syntax and structure |
| 40101 | Missing credentials | Include authentication headers |
| 40102 | Invalid credentials | Verify app ID and secret code |
| 40302 | File too large | Reduce file size or use Parse API first |

See [Errors](/api-reference/endpoint/xparse/v1/errors) for complete error code list.

## Common patterns

### Verify extraction accuracy

```python
result = client.extract.run(file=f, schema=schema)

# Check if required fields were extracted
if result.extracted_data.get("invoice_number") is None:
    print("Warning: Invoice number not found")

# Check citations for a specific field
if "total_amount" in result.citations:
    citation = result.citations["total_amount"]
    print(f"Extracted value: {citation['value']}")
    for region in citation["bounding_regions"]:
        print(f"Found on page {region['page_number']}")
        print(f"Original text: {region['text']}")
```

---

### Build audit trail

```python
# Store extraction with source information
audit_record = {
    "document_id": "doc123",
    "extracted_at": datetime.now(),
    "data": result.extracted_data,
    "sources": {
        field: {
            "value": citation["value"],
            "regions": [
                {
                    "page": region["page_number"],
                    "text": region["text"]
                }
                for region in citation["bounding_regions"]
            ]
        }
        for field, citation in result.citations.items()
    }
}
```

---

### Highlight fields in UI

```python
# Convert citations to UI overlay coordinates
def build_highlights(citations):
    highlights = []
    for field_name, citation in citations.items():
        for region in citation["bounding_regions"]:
            position = region["position"]
            highlights.append({
                "field": field_name,
                "page": region["page_number"],
                "bbox": {
                    "x": position[0],      # Already in pixels
                    "y": position[1],
                    "width": position[2] - position[0],
                    "height": position[5] - position[1]
                }
            })
    return highlights
```
