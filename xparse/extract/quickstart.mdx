Extract pulls specific data from documents using your JSON Schema definition. Get only the fields you need—invoice totals, form data, table rows—without processing the entire document.

## What you'll get

Extract returns structured JSON matching your schema with source citations:

```json
{
  "extracted_data": {
    "invoice_number": "INV-2024-001",
    "total_amount": 1250.00,
    "vendor_name": "Acme Corp"
  },
  "citations": {
    "invoice_number": {
      "page_number": 1,
      "coordinates": [137, 599, 1129, 625],
      "text": "INV-2024-001"
    }
  }
}
```

**Use Extract for:**
- Invoice and receipt processing
- Form data extraction
- Contract metadata extraction
- Structured data that matches your database schema

**When to use Extract vs Parse:**
- **Extract:** You only need specific fields, not full content (faster)
- **Parse:** You need complete document content with preserved structure

## Before you start

Get API credentials from the [TextIn Console](https://www.textin.ai/console/dashboard/setting). See [Authentication](/xparse/authentication) for details.

## Step 1: Define your schema

Create a JSON Schema that defines the fields you want to extract:

```json
{
  "type": "object",
  "properties": {
    "invoice_number": {
      "type": "string",
      "description": "Invoice number"
    },
    "invoice_date": {
      "type": "string",
      "description": "Invoice date"
    },
    "vendor_name": {
      "type": "string",
      "description": "Vendor or merchant name"
    },
    "total_amount": {
      "type": "number",
      "description": "Total amount including tax"
    },
    "items": {
      "type": "array",
      "description": "Invoice line items",
      "items": {
        "type": "object",
        "properties": {
          "description": {
            "type": "string",
            "description": "Item description"
          },
          "quantity": {
            "type": "number",
            "description": "Item quantity"
          },
          "price": {
            "type": "number",
            "description": "Item unit price"
          }
        }
      }
    }
  },
  "required": ["invoice_number", "total_amount", "vendor_name"]
}
```

**Schema components:**
- `type`: Data type at each level (`"object"` at root)
- `properties`: Fields to extract with their definitions
- `description`: Helps AI understand what to extract (important!)
- `required`: Required fields (also defines extraction order)

**Supported types:**
- `string` - Text values
- `number` - Numeric values (int or float)
- `integer` - Integer values only
- `enum` - Predefined choices
- `object` - Nested fields
- `array` - Lists of items

**Allow null values:**
```json
"type": ["string", "null"]  // Field can be string or null
```

<Tip>
  **Use the visual builder:** Create schemas easily with our [web interface](https://www.textin.ai/console/recognition/robot_markdown_beta?service=pdf_to_markdown&tab=llmExtract). Upload a sample document and add fields visually. See [Schema Guide](/xparse/extract/schema-guide) for a walkthrough.
</Tip>

<Warning>
  **Schema limits:**
  - Maximum **3 levels** of nesting (object/array depth)
  - Maximum **100 leaf fields** (fields that hold actual values)
</Warning>

    import json

    # Initialize client

        app_id="your-app-id",
        secret_code="your-secret-code"
    )

    # Load schema
    schema = json.loads(open("invoice_schema.json").read())

    # Extract data
    with open("invoice.pdf", "rb") as f:
        result = client.extract.run(
            file=f,
            schema=schema

    print(result.extracted_data)

## Step 3: Use the results

The API returns structured JSON matching your schema:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "extracted_data": {
      "invoice_number": "INV-2024-001",
      "invoice_date": "2024-01-15",
      "vendor_name": "Acme Corp",
      "total_amount": 1250.00,
      "items": [
        {
          "description": "Product A",
          "quantity": 5,
          "price": 250.00
        }
      ]
    },
    "citations": {
      "invoice_number": {
        "value": "INV-2024-001",
        "bounding_regions": [
          {
            "page_number": 1,
            "position": [137, 599, 1129, 599, 1129, 625, 182, 625],
            "text": "INV-2024-001"
          }
        ]
      }
    }
  }
}
```

**Response includes:**
- `extracted_data`: Structured JSON matching your schema
- `citations`: Source locations with page numbers and coordinates for verification

**Use citations for:**
- Verifying extraction accuracy
- Visual highlighting in your UI
- Audit trails and compliance

## Extraction options

Customize extraction with additional parameters:

```python
result = client.extract.run(
    file=f,
    schema=schema,
    parse_options={
        "crop_dewarp": 1,        # Auto-crop and straighten
        "get_image": "both"      # Extract images: "none", "page", "objects", "both"
    },
    extract_options={
        "generate_citations": True,  # Include source locations
        "stamp": True                # Extract stamp/seal information
    }
)
```

**parse_options** - Document preprocessing:
- `crop_dewarp`: Auto-crop and straighten skewed pages (0 or 1)
- `get_image`: Image extraction mode

**extract_options** - Extraction behavior:
- `generate_citations`: Include source locations (default: true)
- `stamp`: Extract stamp/seal information (default: false)

### Invoice processing example

**Common fields:**
- `invoice_number`, `invoice_date`
- `vendor_name`, `vendor_address`
- `items[]` (description, quantity, unit_price)
- `subtotal`, `tax`, `total`
- `payment_terms`, `due_date`

### Receipt automation example

**Common fields:**
- `merchant_name`, `date`, `time`
- `items[]` (name, price)
- `subtotal`, `tax`, `total`
- `payment_method`, `last_4_digits`

### Form processing example

**Common fields:**
- Personal info: `name`, `email`, `phone`
- Address: `street`, `city`, `state`, `zip_code`
- `checkboxes[]`, `signature_present`
- `date_signed`, `applicant_signature`
