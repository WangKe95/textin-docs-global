We provide an [online web platform](https://www.textin.ai/console/recognition/robot_markdown_beta?service=pdf_to_markdown) that you can use directly through your browser to quickly try our API and experience its capabilities without writing any code.

### Document Parsing

You can click on pre-saved sample documents or upload your own documents (such as invoices, tables, or reports) to quickly view parsing results alongside the original document on the right side. You can switch between JSON format output and specific element parsing results, and perform operations like editing, copying, and exporting the results.

<Warning>
  Please note: Parsing pre-saved sample documents is free, while parsing your own uploaded documents consumes page quota.
</Warning>

![Manual 1 PN](/images/manual-1.PNG)

Parameters affect both parsing results and JSON output content. We provide visual configuration options with some parameters pre-configured. You can customize these parameters according to your actual needs. Once parameter configuration is saved, you can re-parse the document to get the latest results.

![Manual 2 PN](/images/manual-2.PNG)

For quick API integration into your system, we provide convenient code generation functionality that creates sample code based on your current parameter configuration. You can download it with a button click and edit it according to your needs.

### **Document Extraction**

Beyond document parsing, we also provide intelligent document extraction functionality to extract specific data from documents. If you need to extract structured information from multiple documents—for example, extracting invoice information to automate and streamline financial reimbursement processes—this feature will be very useful!

On the [web platform](https://www.textin.ai/console/recognition/robot_markdown_beta?service=pdf_to_markdown), you can switch to "Intelligent Extraction" in the right panel to perform structured information extraction on documents.

**Intelligent extraction supports 2 configuration modes that you can switch between.**

- **Prompt mode**: You can input the content you want to extract, and the system will extract the corresponding information according to your prompt requirements, outputting in JSON format.

<Tip>
  Tips for more stable structure:
  - Use clear field names, such as "transaction amount, signing date"
  - Add simple descriptions to fields, such as "transaction amount (numeric lowercase), signing date (in yy-mm-dd format)"
  - Specify output format, such as "[ date: [detail1: {} ]"
  - Provide clear structured examples
</Tip>

![Manual Prompt Pn](/images/manual-prompt.png)

- **Field mode**: You can customize text fields or table headers to extract, and the system will extract based on these fields. You can view field and header extraction results directly or switch to JSON format to view output. The total number of text fields and table header extraction fields should not exceed 100.

<Tip>
  When defining data to extract, provide a name for each field. You can also add optional descriptions for more context. Both names and descriptions serve as extraction guidance, helping the system more accurately understand what information to find and extract from the document. The more specific and clear the field names and descriptions, the more accurately the system can identify and extract the correct data from documents.
</Tip>

![Manual 5 Pn](/images/manual-5.png)

<Warning>
  Please note: Intelligent document extraction in the API supports documents up to 100 pages maximum, with information beyond that limit being ignored. On the web platform, to help you experience results faster, the maximum is 20 pages.
</Warning>

The online web platform is designed to help you quickly understand what intelligent document parsing can do and experience its capabilities with your own uploaded documents. It's typically not used directly for large-scale production but is more suitable for small-scale testing and evaluation.

For comprehensive document parsing and extraction processing, and integration into your systems, please use our [Document Parsing API](/api-reference/endpoint/xparse/v1/parse-sync) and [Document Extraction API](/api-reference/endpoint/extract-v3).
