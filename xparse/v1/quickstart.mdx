## Using the Python SDK (Recommended)

### Step 1: Install the SDK

<CodeGroup>

```bash pip
pip install xparse-client
```

```bash uv
uv add xparse-client
```

</CodeGroup>

### Step 2: Set Environment Variables

```bash
export TEXTIN_APP_ID="your-app-id"
export TEXTIN_SECRET_CODE="your-secret-code"
```

### Step 3: Parse a Document

```python
from xparse_client import XParseClient, ParseConfig, Capabilities

# Initialize client (automatically reads from environment variables)
client = XParseClient()

# Parse local file
with open("document.pdf", "rb") as f:
    result = client.parse.run(
        file=f,
        filename="document.pdf",
        config=ParseConfig(
            capabilities=Capabilities(
                include_table_structure=True,
                title_tree=True,
            ),
        ),
    )
```

<Tip>
  The example above uses basic configuration. Customize `ParseConfig` for advanced capabilities like character details, inline objects, and image data. See [Parse Configuration](/xparse/v1/parse-config) for details.
</Tip>

### Step 4: View Results

```python
# Output Markdown
if result.markdown:
    print(result.markdown)

# Loop through document elements
for el in result.elements:
    print(f"[{el.type}] {el.text[:80]}")

print(f"Parsed {len(result.elements)} elements")
```

### Step 5: Save Results

```python
import json

# Save as Markdown file
with open("output.md", "w", encoding="utf-8") as f:
    f.write(result.markdown)

# Save full JSON result
with open("output.json", "w", encoding="utf-8") as f:
    json.dump(result.model_dump(), f, ensure_ascii=False, indent=2)
```

### Complete Code (Ready to Run)

```python quickstart.py expandable
"""
Usage:
  1. Install dependencies: pip install xparse-client
  2. Set environment variables TEXTIN_APP_ID and TEXTIN_SECRET_CODE
  3. python quickstart.py <file_path>
"""

import json
import sys
from pathlib import Path

from xparse_client import XParseClient, ParseConfig, Capabilities


def main():
    # Get file path from command line argument, default to document.pdf
    file_path = Path(sys.argv[1]) if len(sys.argv) > 1 else Path("document.pdf")
    if not file_path.exists():
        print(f"File not found: {file_path}")
        sys.exit(1)

    # Step 1: Initialize client (reads TEXTIN_APP_ID, TEXTIN_SECRET_CODE from environment)
    client = XParseClient()

    # Step 2: Parse document
    with open(file_path, "rb") as f:
        result = client.parse.run(
            file=f,
            filename=file_path.name,
            config=ParseConfig(
                capabilities=Capabilities(
                    include_table_structure=True,
                    title_tree=True,
                ),
            ),
        )

    # Step 3: Print parsing summary
    print(f"Parsing complete! {len(result.elements)} elements, {result.success_count} pages successful")
    print("=" * 60)

    # Output Markdown
    if result.markdown:
        print(result.markdown[:2000])
        if len(result.markdown) > 2000:
            print(f"\n... (Markdown output is {len(result.markdown)} characters, truncated)")
    print("=" * 60)

    # Loop through document elements
    for el in result.elements:
        print(f"  [{el.type:20s}] p{el.page_number}: {el.text[:60]}")

    # Step 4: Save results
    output_dir = Path("output")
    output_dir.mkdir(exist_ok=True)

    stem = file_path.stem

    md_path = output_dir / f"{stem}.md"
    with open(md_path, "w", encoding="utf-8") as f:
        f.write(result.markdown)
    print(f"\nMarkdown saved: {md_path}")

    json_path = output_dir / f"{stem}.json"
    with open(json_path, "w", encoding="utf-8") as f:
        json.dump(result.model_dump(), f, ensure_ascii=False, indent=2)
    print(f"JSON saved:     {json_path}")


if __name__ == "__main__":
    main()
```

<Tip>
  For more SDK usage examples, refer to the SDK documentation in your installation package.
</Tip>

## REST API Examples

If you're not using the Python SDK, you can call the [REST API](/api-reference/endpoint/xparse/v1/parse-sync) directly.

### Synchronous Parsing

<CodeGroup>

```python Python
import requests
import json

app_id = "your-app-id"
secret_code = "your-secret-code"

with open("document.pdf", "rb") as f:
    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/sync",
        headers={
            "x-ti-app-id": app_id,
            "x-ti-secret-code": secret_code,
        },
        files={"file": ("document.pdf", f)},
        data={
            "config": json.dumps({
                "capabilities": {
                    "include_table_structure": True,
                    "title_tree": True
                }
            })
        },
    )

result = response.json()
print(result["data"]["markdown"])
```

```bash cURL
curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
  -H "x-ti-app-id: your-app-id" \
  -H "x-ti-secret-code: your-secret-code" \
  -F "file=@document.pdf" \
  -F 'config={"capabilities":{"include_table_structure":true,"title_tree":true}}'
```

</CodeGroup>

<Note>
  The `config` parameter supports rich configuration options including hierarchy relationships, character details, inline objects, and image data. See [Parse Configuration](/xparse/v1/parse-config) for the complete reference.
</Note>

### Async Parsing (For Large Files)

For large files or batch processing scenarios, use the async API:

```python
import requests
import json
import time

app_id = "your-app-id"
secret_code = "your-secret-code"
headers = {
    "x-ti-app-id": app_id,
    "x-ti-secret-code": secret_code,
}

# Step 1: Create async job
with open("large_document.pdf", "rb") as f:
    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/async",
        headers=headers,
        files={"file": ("large_document.pdf", f)},
    )

job_id = response.json()["data"]["job_id"]
print(f"Job created: {job_id}")

# Step 2: Poll for status
while True:
    status_resp = requests.get(
        f"https://api.textin.com/api/v1/xparse/parse/async/{job_id}",
        headers=headers,
    )
    status_data = status_resp.json()["data"]

    if status_data["status"] == "completed":
        print("Parsing complete!")
        print(status_data)
        
        # Step 3: Fetch results via result_url
        result_url = status_data["result_url"]
        result_resp = requests.get(result_url, headers=headers)
        result_data = result_resp.json()
        
        # Output element info
        print(f"Parsed {len(result_data['elements'])} elements")
        for element in result_data["elements"][:5]:
            print(f"[{element['type']}] {element['text'][:50]}")
        break
        
    elif status_data["status"] == "failed":
        print(f"Parsing failed: {status_data.get('message', 'Unknown error')}")
        break

    print(f"Status: {status_data['status']}, waiting...")
    time.sleep(5)
```

<Tip>
  The async API supports `webhook` callbacks to eliminate polling. See the [Async Parsing API Documentation](/api-reference/endpoint/xparse/v1/parse-async) for details.
</Tip>

## Understanding the Response

After successful parsing, the core response structure looks like this:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "schema_version": "1.3.0",
    "file_id": "doc_7f3a2b",
    "job_id": "job_x9k2m",
    "success_count": 5,
    "metadata": {
      "filename": "document.pdf",
      "filetype": "application/pdf",
      "page_count": 5,
      "data_source": {
        "record_locator": {
          "protocol": "file",
          "remote_file_path": "/path/to/document.pdf"
        },
        "url": "file:///path/to/document.pdf"
      }
    },
    "markdown": "# Document Title\n\nBody content...\n\n| Column1 | Column2 |\n|---|---|\n| Value1 | Value2 |",
    "elements": [
      {
        "element_id": "el_001",
        "type": "Title",
        "text": "Document Title",
        "page_number": 1,
        "coordinates": [0.100000, 0.120000, 0.320000, 0.120000, 0.320000, 0.160000, 0.100000, 0.160000],
        "metadata": {
          "category_depth": 0,
          "children_ids": ["el_002"],
          "is_continuation": false,
          "data_source": {
            "record_locator": {
              "protocol": "file",
              "remote_file_path": "/path/to/document.pdf"
            },
            "url": "file:///path/to/document.pdf"
          }
        }
      },
      {
        "element_id": "el_002",
        "type": "NarrativeText",
        "text": "Body content...",
        "page_number": 1,
        "coordinates": [0.100000, 0.180000, 0.900000, 0.180000, 0.900000, 0.220000, 0.100000, 0.220000],
        "metadata": {
          "parent_id": "el_001",
          "is_continuation": false,
          "data_source": {
            "record_locator": {
              "protocol": "file",
              "remote_file_path": "/path/to/document.pdf"
            },
            "url": "file:///path/to/document.pdf"
          }
        }
      }
    ]
  }
}
```

| Field | Description |
|------|------|
| `schema_version` | Schema version number, currently `"1.3.0"` |
| `file_id` | Unique file identifier |
| `job_id` | Unique job identifier |
| `success_count` | Number of successfully parsed pages (billing basis) |
| `metadata` | File metadata (filename, type, page count, data source, etc.) |
| `markdown` | Markdown representation of the document, ready for LLM input |
| `elements` | List of document elements with type, text, coordinates, and metadata |

<Info>
  For detailed response structure documentation, see [Response Structure Reference](/xparse/v1/parse-response).
</Info>
