Parse converts documents into clean, structured data—Markdown for LLMs and JSON elements with coordinates for visual grounding.

## What you'll get

<CardGroup cols={2}>
  <Card title="Markdown" icon="file-lines">
    Clean, AI-ready text with preserved structure—headings, paragraphs, tables, formulas, and reading order.
  </Card>
  
  <Card title="Structured Elements" icon="cube">
    JSON array of document components with types, page numbers, and bounding box coordinates.
  </Card>
</CardGroup>

**Use Parse for:**
- RAG pipelines and vector embeddings
- LLM input (GPT-4, Claude, etc.)
- Search indexing and knowledge bases
- Document Q&A with visual grounding

**Parse vs Extract:**
- **Use Parse** when you need full document content with preserved structure
- **Use Extract** when you only need specific fields (invoice totals, form data, etc.)

## Parse your first document

<Tip>
  **Recommended:** Use the Python SDK for fastest setup.
</Tip>

### Using Python SDK

**Step 1: Install SDK**

```bash
pip install xparse-client
```

**Step 2: Set environment variables**

```bash
export TEXTIN_APP_ID="your-app-id"
export TEXTIN_SECRET_CODE="your-secret-code"
```

**Step 3: Parse a document**

```python
from xparse_client import XParseClient, ParseConfig, Capabilities

# Initialize client (reads environment variables automatically)
client = XParseClient()

# Parse a local file
with open("document.pdf", "rb") as f:
    result = client.parse.run(
        file=f,
        filename="document.pdf",
        config=ParseConfig(
            capabilities=Capabilities(
                include_table_structure=True,
                title_tree=True,
            ),
        ),
    )
```

<Tip>
  This example uses basic configuration. You can customize additional capabilities via `ParseConfig`, such as character details, inline objects, and image data. See [Configuration](/xparse/parse/configuration) for all options.
</Tip>

**Step 4: View results**

```python
# Output Markdown
if result.markdown:
    print(result.markdown)

# Iterate through document elements
for el in result.elements:
    print(f"[{el.type}] {el.text[:80]}")

print(f"Total elements parsed: {len(result.elements)}")
```

**Step 5: Save results**

```python
import json

# Save as Markdown file
with open("output.md", "w", encoding="utf-8") as f:
    f.write(result.markdown)

# Save complete JSON result
with open("output.json", "w", encoding="utf-8") as f:
    json.dump(result.model_dump(), f, ensure_ascii=False, indent=2)
```

**Complete code (ready to run)**

```python quickstart.py expandable
"""
Usage:
  1. Install dependencies: pip install xparse-client
  2. Set environment variables TEXTIN_APP_ID and TEXTIN_SECRET_CODE
  3. python quickstart.py <file_path>
"""

import json
import sys
from pathlib import Path

from xparse_client import XParseClient, ParseConfig, Capabilities


def main():
    # Get file path from command line argument, default to document.pdf
    file_path = Path(sys.argv[1]) if len(sys.argv) > 1 else Path("document.pdf")
    if not file_path.exists():
        print(f"File not found: {file_path}")
        sys.exit(1)

    # Step 1: Initialize client (reads TEXTIN_APP_ID and TEXTIN_SECRET_CODE from environment)
    client = XParseClient()

    # Step 2: Parse document
    with open(file_path, "rb") as f:
        result = client.parse.run(
            file=f,
            filename=file_path.name,
            config=ParseConfig(
                capabilities=Capabilities(
                    include_table_structure=True,
                    title_tree=True,
                ),
            ),
        )

    # Step 3: Print parsing summary
    print(f"Parsing complete! {len(result.elements)} elements from {result.success_count} pages")
    print("=" * 60)

    # Output Markdown
    if result.markdown:
        print(result.markdown[:2000])
        if len(result.markdown) > 2000:
            print(f"\n... (Markdown has {len(result.markdown)} characters, truncated)")
    print("=" * 60)

    # Iterate through document elements
    for el in result.elements:
        print(f"  [{el.type:20s}] p{el.page_number}: {el.text[:60]}")

    # Step 4: Save results
    output_dir = Path("output")
    output_dir.mkdir(exist_ok=True)

    stem = file_path.stem

    md_path = output_dir / f"{stem}.md"
    with open(md_path, "w", encoding="utf-8") as f:
        f.write(result.markdown)
    print(f"\nMarkdown saved: {md_path}")

    json_path = output_dir / f"{stem}.json"
    with open(json_path, "w", encoding="utf-8") as f:
        json.dump(result.model_dump(), f, ensure_ascii=False, indent=2)
    print(f"JSON saved:     {json_path}")


if __name__ == "__main__":
    main()
```

<Note>
  For large documents or batch processing, use the [Async API](/xparse/parse/async-processing) instead of the sync endpoint.
</Note>

### Using REST API

If you don't want to use the Python SDK, call the [REST API](/api-reference/endpoint/xparse/v1/parse-sync) directly.

**Synchronous parsing**

<CodeGroup>

```python Python
import requests
import json

app_id = "your-app-id"
secret_code = "your-secret-code"

with open("document.pdf", "rb") as f:
    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/sync",
        headers={
            "x-ti-app-id": app_id,
            "x-ti-secret-code": secret_code,
        },
        files={"file": ("document.pdf", f)},
        data={
            "config": json.dumps({
                "capabilities": {
                    "include_table_structure": True,
                    "title_tree": True
                }
            })
        },
    )

result = response.json()
print(result["data"]["markdown"])
```

```bash cURL
curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
  -H "x-ti-app-id: your-app-id" \
  -H "x-ti-secret-code: your-secret-code" \
  -F "file=@document.pdf" \
  -F 'config={"capabilities":{"include_table_structure":true,"title_tree":true}}'
```

</CodeGroup>

<Note>
  The `config` parameter supports extensive configuration options including hierarchy, character details, inline objects, and image data. See [Configuration](/xparse/parse/configuration) for all options.
</Note>

**Asynchronous parsing (for large files)**

For large documents (over 500 pages) or batch processing, use the async API:

```python
import requests

app_id = "your-app-id"
secret_code = "your-secret-code"

# Create async job
with open("large_document.pdf", "rb") as f:
    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/async",
        headers={
            "x-ti-app-id": app_id,
            "x-ti-secret-code": secret_code,
        },
        files={"file": ("large_document.pdf", f)},
    )

job_id = response.json()["data"]["job_id"]
print(f"Job created: {job_id}")

# Poll status and download result - see Async Processing guide
```

<Info>
  **Learn more:** See [Async Processing](/xparse/parse/async-processing) for complete examples including status polling, webhook callbacks, batch processing, and error handling.
</Info>

## Understanding the response

A successful parse returns the following data structure:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "schema_version": "1.3.0",
    "file_id": "doc_7f3a2b",
    "job_id": "job_x9k2m",
    "success_count": 5,
    "metadata": {
      "filename": "document.pdf",
      "filetype": "application/pdf",
      "page_count": 5,
      "data_source": {
        "record_locator": {
          "protocol": "file",
          "remote_file_path": "/path/to/document.pdf"
        },
        "url": "file:///path/to/document.pdf"
      }
    },
    "markdown": "# Document Title\n\nThis is the content...\n\n| Col1 | Col2 |\n|---|---|\n| Val1 | Val2 |",
    "elements": [
      {
        "element_id": "el_001",
        "type": "Title",
        "text": "Document Title",
        "page_number": 1,
        "coordinates": [0.100000, 0.120000, 0.320000, 0.120000, 0.320000, 0.160000, 0.100000, 0.160000],
        "metadata": {
          "category_depth": 0,
          "children_ids": ["el_002"],
          "is_continuation": false,
          "data_source": {
            "record_locator": {
              "protocol": "file",
              "remote_file_path": "/path/to/document.pdf"
            },
            "url": "file:///path/to/document.pdf"
          }
        }
      },
      {
        "element_id": "el_002",
        "type": "NarrativeText",
        "text": "This is the content...",
        "page_number": 1,
        "coordinates": [0.100000, 0.180000, 0.900000, 0.180000, 0.900000, 0.220000, 0.100000, 0.220000],
        "metadata": {
          "parent_id": "el_001",
          "is_continuation": false,
          "data_source": {
            "record_locator": {
              "protocol": "file",
              "remote_file_path": "/path/to/document.pdf"
            },
            "url": "file:///path/to/document.pdf"
          }
        }
      }
    ]
  }
}
```

| Field | Description |
|-------|-------------|
| `schema_version` | Data structure version (currently `"1.3.0"`) |
| `file_id` | Unique identifier for the file |
| `job_id` | Unique identifier for the parsing job |
| `success_count` | Number of pages successfully parsed (billing basis) |
| `metadata` | File metadata including filename, type, page count, and data source |
| `markdown` | Markdown representation of the document (LLM-ready) |
| `elements` | Array of document elements with type, text, coordinates, and metadata |

  See [Response Format](/xparse/parse/response-format) for complete documentation of the response structure.
</Info>
