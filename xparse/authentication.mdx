TextIn API requests are authenticated with an App ID and Secret Code sent in the request headers:

```
x-ti-app-id: YOUR_APP_ID
x-ti-secret-code: YOUR_SECRET_CODE
```

## Get your credentials

<Steps>
  <Step title="Open the console">
    Go to [Account & Developer Info](https://www.textin.ai/console/dashboard/setting) in the TextIn console.
  </Step>

  <Step title="Locate your credentials">
    Sign in to the TextIn console, click your profile icon in the bottom-left corner, and select **Account & Developer Info**.

    ![AP Ikey 1 Pn](/images/APIkey-1.png)
  </Step>

  <Step title="Copy your credentials">
    Click the **Copy** icon to copy your `x-ti-app-id` and `x-ti-secret-code`.

    ![AP Ikey Jp](/images/APIkey.jpg)
  </Step>
</Steps>

## Add credentials to a request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
      -H "x-ti-app-id: $TEXTIN_APP_ID" \
      -H "x-ti-secret-code: $TEXTIN_SECRET_CODE" \
      -F "file=@document.pdf"
    ```
  </Tab>

  <Tab title="Python">
    ```python
    import requests

    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/sync",
        headers={
            "x-ti-app-id": "your-app-id",
            "x-ti-secret-code": "your-secret-code"
        },
        files={"file": open("document.pdf", "rb")}
    )
    ```
  </Tab>

  <Tab title="Python SDK">
    The SDK automatically reads from environment variables:

    ```python
    from xparse_client import XParseClient

    # Reads TEXTIN_APP_ID and TEXTIN_SECRET_CODE from environment
    client = XParseClient()
    ```

    Set environment variables:

    ```bash
    export TEXTIN_APP_ID="your-app-id"
    export TEXTIN_SECRET_CODE="your-secret-code"
    ```
  </Tab>
</Tabs>

## Verify your credentials

Test your credentials with a simple request:

```bash
curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
  -H "x-ti-app-id: YOUR_APP_ID" \
  -H "x-ti-secret-code: YOUR_SECRET_CODE" \
  -F "file_url=https://dllf.intsig.net/download/2025/Solution/textin/sample/pdf_to_markdown/sample_02.pdf"
```

A successful response returns `"code": 200` with parsed document data. If you see an error, check the [Troubleshooting](#troubleshooting) section below.

## Store credentials securely

- **Don't commit** credentials to Git or version control
- **Don't expose** them in browser or client-side code
- **Use environment variables** or a secret manager in production
- **Regenerate credentials** if you suspect a leak (contact support for help)

## Troubleshooting

| Error Code | Description | Solution |
|------------|-------------|----------|
| 40101 | `x-ti-app-id` or `x-ti-secret-code` is missing | Include both headers in your request |
| 40102 | `x-ti-app-id` or `x-ti-secret-code` is invalid | Verify your credentials in the [console](https://www.textin.ai/console/dashboard/setting) |
| 40103 | Client IP not in whitelist | Add your IP to the whitelist in console settings |

<Tip>
  If you continue to see authentication errors after verifying your credentials, contact [TextIn support](https://www.textin.ai/contact) for assistance.
</Tip>
