## How authentication works

Include your App ID and Secret Code in every API request:

```
x-ti-app-id: YOUR_APP_ID
x-ti-secret-code: YOUR_SECRET_CODE
```

## Get your credentials

<Steps>
  <Step title="Sign in to the TextIn console">
    Sign in to your [TextIn account](https://www.textin.ai/console/dashboard/setting).
  </Step>

  <Step title="Open Account & Developer Info">
    Click your profile icon in the bottom-left corner, then select **Account & Developer Info**.

    ![AP Ikey 1 Pn](/images/APIkey-1.png)
  </Step>

  <Step title="Copy your credentials">
    Copy your App ID and Secret Code. Use them in the `x-ti-app-id` and `x-ti-secret-code` request headers.

    ![AP Ikey Jp](/images/APIkey.jpg)
  </Step>
</Steps>

## Verify your credentials

Send a test request:

```bash
curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
  -H "x-ti-app-id: YOUR_APP_ID" \
  -H "x-ti-secret-code: YOUR_SECRET_CODE" \
  -F "file_url=https://..."
```

If authentication succeeds, the response body includes `"code": 200` and the parsed document data.

## Use environment variables

To avoid hardcoding credentials, store them in environment variables.

**macOS and Linux**
```bash
export TEXTIN_APP_ID="your-app-id"
export TEXTIN_SECRET_CODE="your-secret-code"
```

**Windows PowerShell**
```powershell
$env:TEXTIN_APP_ID="your-app-id"
$env:TEXTIN_SECRET_CODE="your-secret-code"
```

The following examples read credentials from these environment variables.

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.textin.com/api/v1/xparse/parse/sync" \
      -H "x-ti-app-id: $TEXTIN_APP_ID" \
      -H "x-ti-secret-code: $TEXTIN_SECRET_CODE" \
      -F "file=@document.pdf"
    ```
  </Tab>

  <Tab title="Python (SDK)">
    ```python
    from xparse_client import XParseClient

    # Automatically reads TEXTIN_APP_ID and TEXTIN_SECRET_CODE
    client = XParseClient()
    ```
  </Tab>

  <Tab title="Python (requests)">
    ```python
    import os
    import requests

    response = requests.post(
        "https://api.textin.com/api/v1/xparse/parse/sync",
        headers={
            "x-ti-app-id": os.getenv("TEXTIN_APP_ID"),
            "x-ti-secret-code": os.getenv("TEXTIN_SECRET_CODE")
        },
        files={"file": open("document.pdf", "rb")}
    )
    ```
  </Tab>
</Tabs>

## Keep credentials secure

- Never commit credentials to version control.
- Keep credentials on the server. Do not expose them in client-side code.
- Use environment variables or a secret manager in production.
- Contact TextIn support immediately if you believe your credentials have been exposed.

## Troubleshooting

| Error Code | Description | Solution |
|------------|-------------|----------|
| 40101 | `x-ti-app-id` or `x-ti-secret-code` is missing | Include both headers in your request |
| 40102 | `x-ti-app-id` or `x-ti-secret-code` is invalid | Verify your credentials in the [console](https://www.textin.ai/console/dashboard/setting) |
| 40103 | Client IP not in IP allowlist | Add your IP to the IP allowlist in console settings |

<Tip>
  If you continue to see authentication errors after verifying your credentials, contact [TextIn support](https://www.textin.ai/contact) for assistance.
</Tip>
